# Astro Project Rules

## Core Rules

- Use React with Next.js App Router.
- Avoid inline styles whenever there is a clean CSS alternative.
- Keep code modular, reusable, and componentized.
- Avoid duplicated or redundant code.
- Keep files focused and reasonably small.
- Prefer shared utilities, shared section patterns, and data-driven rendering.
- Keep code clean, well-structured, and lightly commented only where useful.
- Do not use Bootstrap or Tailwind.
- Build with SEO in mind from the start.
- Prefer efficient implementations and avoid unnecessary package or token usage.

## Styling Rules

- Use CSS Modules and shared global tokens/utilities.
- Centralize design tokens like colors, spacing, shadows, and transitions.
- Reuse animation patterns instead of redefining them per component.
- Favor semantic HTML and accessible interactions.

## Content Rules

- Use the provided Astro logo and hero video from local assets.
- Recreate the provided homepage preview faithfully while replacing placeholders with production-ready content or visuals.
- Pull website copy and product data from the current Astro Mining website when applicable.

## Documentation Rules

- Maintain `README.md` as the project evolves.
- Document architecture, setup, and major implementation decisions.
- After any completed code change, commit and push the update.
